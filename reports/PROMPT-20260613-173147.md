## Diagnosis
The churn model evaluation relied on accuracy (0.93) as the primary metric on a severely imbalanced 90/10 dataset. This produced a misleadingly high headline number while the minority-class F1 collapsed from a baseline of 0.65 to 0.14, indicating the XGBoost classifier is heavily biased toward the majority class and missing nearly all churners. The chosen validation strategy (plain 80/20 train-test split) did not guarantee representative minority-class proportions in the hold-out set, violating standard practice for imbalanced classification.
- Violated assumption: That accuracy is a sufficient and reliable performance indicator and that a default (non-stratified) train-test split will produce trustworthy estimates on skewed class distributions.
- Knowledge gap: The team is unaware of DP-100 guidance on imbalanced classification: accuracy should be de-emphasized in favor of minority recall, F1, precision, confusion-matrix shape, and stratified sampling (or time-aware splits when order matters). They also appear unaware that XGBoost requires explicit class weighting (scale_pos_weight) or resampling when the positive class is rare.
- Evidence strength: strong
