## Diagnosis
The XGBoost classifier on a 90/10 churn dataset achieved 93% accuracy by correctly classifying the vast majority class while almost completely failing on the minority churn class (minority F1 fell from 0.65 baseline to 0.14). This is an evaluation methodology failure: accuracy was used as the headline metric despite the severe class imbalance, exactly as warned in the DP-100 skill guide. No drift, data-quality, or leakage signals were present, confirming the root issue lies in metric selection and threshold evaluation rather than data or modeling per se.
- Violated assumption: That accuracy is a sufficient and reliable performance metric for an imbalanced business problem (customer churn) where the operationally critical class is rare.
- Knowledge gap: The team treated accuracy as the primary success criterion without applying imbalance-aware evaluation (minority F1, recall, precision, confusion-matrix shape, or cost-sensitive thresholds) required by DP-100 style ML practice and the failure taxonomy.
